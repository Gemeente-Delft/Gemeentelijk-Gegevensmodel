# Mobility

The data definitions for Mobility connect to those of Public Space. Several object types from the [Information Model Public Space Management (IMBOR)](https://www.crow.nl/thema-s/management-openbare-ruimte/imbor) are reused for traffic. These are the management objects:

1. *Pole* (traffic light is a specialisation of *Pole*);
2. *Sensor* (the induction loops in the road surface are a specialisation of *Sensor*);
3. *Cabinet* (traffic control installations are a specialisation of *Cabinet*).

The above object types are elaborated in the [Public Space](ruimteAlgemeen.en.md#management-objects) section of the GGM. The next figure shows how *Traffic Count* and *V-log-info* are generated by these three object types.

![Traffic Counts data model][tellingen]

<em>Diagram (in Dutch): data model for Traffic Counts, combining Pole, Sensor and Cabinet management objects.</em>

The attributes of V-log-info and Traffic Count show that V-log-info can contain more information. The next figure shows the different attributes of the V-log object types.

![V-Log attributes][vlog]

<em>Diagram (in Dutch): attributes of V-Log objects.</em>

Road closures are shown in the next figure. The object type *Closure* concerns a number of road sections and is entered or modified by a *Staff Member*. Each *Closure* has characteristics:

1. Nuisance category;
2. Number of people hindered;
3. Nuisance class;
4. Closure status.

![Traffic Closures][stremmingen]

<em>Diagram (in Dutch): data model for traffic closures.</em>

Traffic decisions are recorded in a *Document*.
